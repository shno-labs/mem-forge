# Keep Source List organization personal and orthogonal to Projects

Project binding remains the semantic destination and retrieval grouping for a Source, while pinning, sorting, and searching belong to each user's Source List View. A Pinned Source stays inside its Project and is rendered before the selected sort order; search only narrows the current view. We deliberately reject a second Collection hierarchy, global duplicate pinned rows, and shared manual ordering because they would blur retrieval semantics with personal navigation and create conflicting team-wide state.
